
Completely rebuild the Plumbum web app from scratch. Keep the existing FastAPI backend and /api/risk endpoint logic untouched. Rebuild only the frontend as a multi-page Next.js application with a design quality matching The New York Times, ProPublica, or The Marshall Project. Every page must feel like it was built by a senior design team with months of polish.
Global design system — apply everywhere:
Font: Playfair Display for all headlines and the logo. Inter for all body text, labels, nav links. Load both from Google Fonts.

Colors: Background #F8F6F1, text #1A1A18, accent #A63D2F brick red, surface #EDEAE3, dark #0F0F0D, gray #888880.

Spacing: Every section minimum 96px padding top and bottom. Never let sections touch each other. Generous whitespace is the design.

Borders: 1px or 1.5px solid lines only — no box shadows, no cards with drop shadows, no rounded corners above 4px.

Typography scale: 72px display, 48px h1, 36px h2, 24px h3, 18px body large, 14px body, 11px label all-caps letter-spacing 0.12em.

No gradients. No shadows. No illustrations. No stock photos. Data and typography are the only visual elements.

Technical requirements:
Next.js 14 app router. All pages server-side rendered where possible. FastAPI backend unchanged, proxied through Next.js API routes. Leaflet.js maps loaded client-side only with dynamic imports and ssr:false. All mock data lives in /lib/mockData.ts so it can be swapped for real model output in one place. CSS modules for all styling — no Tailwind, no component libraries. Fully mobile responsive — every page stacks gracefully at 375px. Page transitions: 200ms opacity fade between routes. No layout shift on font load — use font-display: swap and set explicit dimensions on all map containers.
The codebase must be clean enough that a researcher with no frontend experience can read it and understand exactly what each file does. Every folder has a README.md with one paragraph explaining its purpose.
