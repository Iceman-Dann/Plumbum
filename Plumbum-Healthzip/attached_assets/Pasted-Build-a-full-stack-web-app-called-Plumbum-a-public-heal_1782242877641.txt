Backend (/api/risk?address=...)
FastAPI endpoint that takes an address string, geocodes it using the free Census Geocoder API (https://geocoding.geo.census.gov/geocoder), then returns a mock risk score object with this shape:
json{
  "address": "123 Main St, Newark NJ",
  "score": 74,
  "risk_level": "High",
  "factors": [
    "Housing built before 1940",
    "EPA violations recorded 2018",
    "Elevated soil lead levels in tract"
  ],
  "lat": 40.7357,
  "lng": -74.1724
}
For now the score is calculated as a simple deterministic function of the geocoded census tract FIPS code so every address returns a consistent result. The real ML model will be swapped in later — just make the architecture clean and modular so replacing the scoring function is one file change.
