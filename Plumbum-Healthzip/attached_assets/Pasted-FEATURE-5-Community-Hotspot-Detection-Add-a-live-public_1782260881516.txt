FEATURE 5 — Community Hotspot Detection
Add a live public hotspot map at /hotspots showing which census tracts are being searched most frequently, creating a real-time picture of where communities are most concerned about lead risk.
Backend: add a searches table to a SQLite database using SQLAlchemy. Every time GET /api/risk is called, log the census tract FIPS code, the risk score, the timestamp, and a randomly generated anonymous session ID — never log the actual address. Add a new endpoint GET /api/hotspots that returns census tracts with more than 5 searches in the past 30 days, ranked by search count, including the average risk score for that tract.
Frontend /hotspots page: same nav. Header section with 11px all-caps brick red label "LIVE DATA" and headline "Where communities are looking" in 56px Playfair bold. Subtext: "Every search on Plumbum is anonymously logged by census tract. When a community searches repeatedly, it signals concern. This map shows where people are looking — and what they're finding."
Full-width Leaflet map 600px tall using the dark CartoDB basemap. Each census tract that appears in the hotspot data is drawn as a colored polygon — the fill color maps to average risk score from light gray to brick red, and the polygon opacity maps to search volume so heavily searched tracts appear more saturated. Clicking a tract polygon shows a popup: tract name, search count in the past 30 days, average risk score, and a "View tract data →" link.
Below the map a ranked table: columns are Rank, Census Tract, City, Searches (30 days), Average Risk Score. Sortable by clicking column headers. Each row links to /result?address= with the tract centroid coordinates.
Below the table add a "Media and research access" section: "Journalists and researchers can download the full anonymized hotspot dataset as CSV. Updated daily." A plain download button. This is what gets Plumbum covered by local news.
At the bottom add a privacy statement in small gray text: "No addresses are stored. No personal data is collected. Search logs contain only census tract FIPS codes, timestamps, and anonymous session IDs. Full privacy policy at plumbum.io/privacy."
